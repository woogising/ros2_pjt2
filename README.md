# A-4 ros2_prj2

### od_msg(제목 수정 필요) : 인터페이스

현재 코드는 전체 흐름 기준으로 0~5번까지는 거의 구현됨.

6번은 “정리해줘” 명령까지는 구현됨.

7번 실제 Pick & Place는 아직 미구현.

8번 재검증 흐름은 코드상 연결되어 있지만 7번이 되어야 실제로 의미 있음.

9번 완료 보고는 템플릿 기반으로 일부 구현됐고, VLM 기반 판단은 아직 미구현.
<br />

<<전체 동작 흐름>> 

전제: robot_control_node / realsense_node는 계속 실행되고 있음

0. 로봇 동작 전 대기상태

1. 사용자가 로봇을 음성으로 호출
	STT + LLM으로 동작 시작 프로세스 호출
	예: “로봇아, 작업공간 확인해줘.”

2. 로봇이 카메라로 작업영역을 스캔
	RGB + Depth Camera로 작업영역을 촬영
	RGB 이미지에서 YOLO/YOLO-Seg를 통해 물체의 종류, 위치, 마스크 정보를 인식
	Depth Camera를 이용해 각 물체 중심점 또는 마스크 영역의 깊이값을 추출
	인식된 물체별로 class, 중심 좌표, depth, confidence 정보를 생성

3. 물체의 실제 위치와 소속 구역을 판단
	카메라 이미지 좌표와 Depth 정보를 이용해 물체의 3D 위치를 추정
	카메라 좌표계를 로봇 기준 좌표계 또는 작업공간 기준 좌표계로 변환
	변환된 X, Y 좌표를 기준으로 물체가 A/B/C/D 중 어느 구역에 놓여 있는지 판정
	각 물체에 대해 현재 구역, 올바른 구역, 오배치 여부를 데이터로 정리

4. 구역 규칙과 현재 상태 비교
	사전에 정의된 구역별 물품 규칙을 불러옴(예: A구역은 과일, B구역은 공구, C구역은 필기구, D구역은 박스에 있어야 함)
	3번 단계에서 정리된 각 물체의 현재 구역과 올바른 구역을 비교
	규칙과 일치하지 않는 물체를 오배치 물체로 분류
	오배치 물체별로 현재 위치, 이동해야 할 목표 구역, 우선순위를 정리

5. 잘못 배치된 물건 개수와 종류를 음성으로 안내
	4번 단계에서 생성된 오배치 물체 목록을 기반으로 안내 문장 생성
	LLM 또는 문장 템플릿을 사용해 사용자가 이해하기 쉬운 자연어로 변환
	TTS를 통해 음성으로 안내
	예: “현재 잘못 배치된 물건은 3개입니다. 공구 1개와 필기구 2개가 다른 구역에 있습니다.”

6. 사용자가 작업 지시
	STT를 통해 사용자의 음성 명령을 텍스트로 변환
	LLM을 사용해 사용자의 명령 의도를 해석
	예: “정리해줘.” / “내가 할게.”
	사용자 명령이 “정리해줘”인 경우 로봇 정리 작업을 수행
	사용자 명령이 “내가 할게”인 경우 알겠다는 음성 응답 후 0번 대기 상태로 복귀

7. 로봇이 오배치 물체를 올바른 구역으로 이동 ("내가 할게"의 경우 알겠다는 대답과 함께 0으로)
	4번 단계에서 정리된 오배치 물체 목록을 기준으로 이동 순서를 결정
	각 오배치 물체의 현재 위치와 목표 구역 정보를 불러옴
	YOLO/YOLO-Seg와 Depth Camera 기반으로 계산된 3D 좌표를 사용해 Pick & Place 수행
	물체를 현재 위치에서 집어 올린 뒤, 해당 물체의 올바른 목표 구역으로 이동
	오배치 물체 목록의 모든 물체에 대해 동일한 과정을 반복 수행

-> 동일한 과정을 반복수행하는걸 어떻게 하면 좋을지 생각하기. 흠...

8. 작업 완료 후 재검증
	로봇의 정리 작업이 끝난 뒤 카메라로 작업영역을 다시 촬영
	YOLO/YOLO-Seg로 물체를 재인식하고 Depth Camera로 위치를 다시 추정
	좌표 기반으로 각 물체가 A/B/C/D 중 어느 구역에 놓여 있는지 재판정
	사전에 정의된 구역 규칙과 비교하여 오배치 물체가 남아 있는지 확인
	오배치 물체가 남아 있으면 실패 또는 추가 정리 대상으로 분류
	오배치 물체가 없으면 작업 완료 상태로 판단

9. 음성으로 완료 보고
	VLM에 입력할 것
		- 재검증 카메라 이미지
		- A/B/C/D 구역 정보
		- 구역별 규칙
		- YOLO/YOLO-Seg가 인식한 물체 목록
		- 각 물체의 현재 구역
		- 오배치 후보 리스트
	VLM을 사용해 현재 작업공간 상태를 “정상 완료”, “오배치 물체 남음”, “판단 불확실” 중 하나로 분류
	오배치 물체가 없는 경우 작업 완료 메시지를 생성
		예: “정리가 완료되었습니다. 모든 물건이 지정된 구역에 배치되었습니다.”
	오배치 물체가 남아 있는 경우 남은 물체의 종류와 위치를 사용자에게 설명
		예: “정리 작업을 완료했지만, 필기구 1개가 아직 공구 구역에 남아 있습니다.”
	판단이 불확실한 경우 사용자에게 확인 요청
		예: “일부 물체가 겹쳐 있어 정확한 배치 여부를 판단하기 어렵습니다. 확인이 필요합니다.”
	생성된 자연어 문장은 TTS를 통해 음성으로 출력
